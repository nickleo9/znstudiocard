🧂 廚房基本配備對照
🍳
Canvas 畫布
廚房
做所有事的地方，你的主要工作空間
🧂
Node 節點
食材 / 廚具
每個烹飪動作：切、炒、煮、調味，各有功能
📦
節點庫
冰箱 + 調味料架
所有可用的食材工具都在這裡等你取用
🔥
Trigger 觸發
點火 / 開始下鍋
啟動整個料理流程的那個關鍵動作
🥦
Data 資料
食材本身
在每個步驟之間流動、被加工處理的東西
➡️
Connection 連線
傳菜順序
炒好→裝盤→上桌，有先後邏輯不能亂
👅
IF 條件節點
試味道
「夠鹹了嗎？」是→上桌，否→繼續加鹽
🔄
Loop 迴圈
批次備料
同一個手法重複用在每一塊食材上
🚚
API
食材供應商
不用自己種，跟外部叫貨就好用
🪪
Credentials
廚師證 / 會員帳號
你能進倉庫取貨的身份憑證
🥣
Sub-workflow
預製半成品
醃好的肉、熬好的高湯，隨時取用
🧯
Error 錯誤處理
燒焦的補救方案
出錯了怎麼辦？預先設好應對措施
🍅 實戰示範：番茄炒蛋 = 你的第一條工作流
🔥
Trigger 觸發節點
開火 — 手動觸發 / 排程
按下執行，或設定時間自動啟動整個流程
🌐
HTTP Request 節點
去市場買番茄 — 呼叫 API 取得資料
從外部來源（API、表單、資料庫）把食材帶回來
🔪
Set / Code 節點
切菜打蛋 — 整理與處理資料
把取得的資料清洗、轉換成你需要的格式
👅
IF 條件節點
試味道 — 判斷資料是否符合條件
符合條件走 A 路線，不符合走 B 路線
🍽️
Send Email / LINE 節點
上菜 — 發送結果給使用者
把處理完的成果推送到 Email、LINE、Slack 等地方
